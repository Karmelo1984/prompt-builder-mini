AI Prompt & Workflow Builder

En V1 se generan prompts optimizados para GPT, Claude y Claude Code sin mezclar decisiones, contexto, restricciones ni formato de salida.

Paso 1

¿Cuál es tu rol o perfil?

Sin seleccionar

Elige un perfil o rol. Este paso inicial reinicia la plantilla, contexto, restricciones y salida que ya tengas marcadas.

Regla de ahorro

El objetivo no es escribir el prompt más corto, sino evitar varias rondas de correcciones. En conjunto, un prompt estructurado suele gastar menos tokens totales.

Paso 2

Plantilla

Sin seleccionar

Elige la plantilla exacta para tu caso de uso. Cambiarla borra el contexto escrito y las selecciones posteriores.

Paso 3

Rellena el contexto mínimo

Pega solo lo necesario: stack, objetivo, porqué y el input verificable. Si modificas este paso después de marcar restricciones o salida, esas selecciones se limpian.

Paso 4

Restricciones

Las restricciones impiden que la IA “ayude” reescribiendo medio proyecto cuando solo querías cambiar una bombilla.

Paso 5

Salida requerida

Pide una salida concreta: causa, diff, tests y verificación. Lo contrario genera literatura técnica decorativa.
